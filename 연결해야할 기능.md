프론트에서 필요한 API 응답을 정리했습니다.

1. 지역정보 목록

`GET /api/locations?category=&search=&page=1&size=20`

받을 값:
- 장소 ID
- 장소명
- 카테고리
- 주소
- 위도·경도
- 이미지
- 전체 데이터 수
- 현재 페이지
- 전체 페이지 수

```json
{
  "items": [],
  "total": 100,
  "page": 1,
  "size": 20,
  "totalPages": 5
}
```

2. 지도 마커

`GET /api/locations/map?category=tourist`

받을 값:
- 장소 ID
- 장소명
- 카테고리
- 주소
- 위도·경도

3. 게시글 목록·검색

`GET /api/posts?category=&search=&page=1&size=10`

받을 값:
- 게시글 ID
- 카테고리
- 제목
- 작성자
- 내용
- 작성일
- 전체 게시글 수
- 전체 페이지 수

검색은 제목과 내용에만 적용하고 조회수는 필요 없습니다.

4. 게시글 상세

`GET /api/posts/{id}`

받을 값:
- 게시글 전체 내용
- 해당 게시글의 댓글 목록

비밀번호는 응답에 포함하지 않습니다.

5. 게시글 작성·수정

`POST /api/posts`
`PUT /api/posts/{id}`

성공하면 생성 또는 수정된 게시글 전체와 게시글 ID를 반환해 주세요.

6. 게시글·댓글 삭제

`DELETE /api/posts/{id}`
`DELETE /api/posts/{id}/comments/{commentId}`

성공하면 `204 No Content`로 반환해 주세요.

7. 댓글 작성

`POST /api/posts/{id}/comments`

성공하면 생성된 댓글 ID, 작성자, 내용, 작성일을 반환해 주세요.

8. 챗봇

`POST /api/chat`

```json
{
  "message": "챗봇 답변"
}
```

형태로 반환해 주세요.

9. 오류 응답

모든 오류는 아래 형식으로 통일해 주세요.

```json
{
  "detail": "오류 메시지"
}
```

그리고 CORS에 아래 프론트 주소를 허용해 주세요.

`http://localhost:5500`

백엔드 실행 주소는 우선 `http://localhost:8000`으로 연결할 예정입니다.
