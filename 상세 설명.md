# LocalHub 화면·API 연동 명세서

- 문서 버전: 1.0.0
- 작성일: 2026-07-15
- 대상 지역: 서울
- 프론트엔드: Vue.js 3 CDN SPA
- 백엔드: FastAPI
- 데이터베이스: SQLite + SQLAlchemy
- 제공 데이터: 한국관광공사 TourAPI 4.0 가공 JSON

## 1. 시스템 동작 구조

```text
사용자 클릭
→ Vue Router로 화면 이동
→ 이동한 Vue 화면에서 FastAPI 호출
→ FastAPI가 SQLite 데이터 조회·등록·수정·삭제
→ JSON 응답
→ Vue 화면에 결과 표시
```

프론트엔드는 SQLite에 직접 접근하지 않는다. 모든 데이터는 FastAPI REST API를 통해 처리한다.

개발 환경 기본 주소:

```text
프론트엔드: http://localhost:5500
백엔드: http://localhost:8000
API 기본 경로: http://localhost:8000/api
```

프론트 설정:

```js
window.LOCALHUB_CONFIG = {
  API_BASE_URL: 'http://localhost:8000',
  USE_MOCK_API: false,
  KAKAO_MAP_KEY: '카카오_JavaScript_키',
}
```

실제 ZIP의 서울 폴더에는 다음 7개 JSON 파일이 존재한다.

|---|---:|---|---:|

| 레포츠 | 28 | `sports` | 126 |
| 문화시설 | 14 | `culture` | 566 |
| 쇼핑 | 38 | `shopping` | 4,368 |
| 숙박 | 32 | `stay` | 423 |
| 여행코스 | 25 | `course` | 51 |
| 축제공연행사 | 15 | `festival` | 201 |
| 합계 |  |  | 6,518 |

`SOURCE.md`에는 음식점 1,632건을 포함하여 총 8,150건이라고 기재되어 있지만, 실제 서울 ZIP 폴더에는 음식점 JSON 파일이 없다. 현재 서비스는 실제 파일과 일치하는 7개 카테고리를 사용한다. 음식점 기능을 추가하려면 누락된 파일을 별도로 제공받아야 한다.

## 3. 원본 JSON 구조와 API 변환

원본 파일 최상위 구조:

```json
{
  "region": "서울",
  "contentType": "관광지",
  "contentTypeId": "12",
  "total": 783,
  "items": []
}
```

원본 항목 예시:

```json
{
  "contentid": "1059877",
  "contenttypeid": "12",
  "title": "양화한강공원",
  "addr1": "서울특별시 영등포구 노들로 221 (당산동)",
  "addr2": "",
  "zipcode": "",
  "tel": "",
  "mapx": "126.8971",
  "mapy": "37.5385",
  "firstimage": "https://example.com/image.jpg",
  "firstimage2": "https://example.com/thumbnail.jpg",
  "cpyrhtDivCd": "Type3"
}
```

프론트 응답 권장 형식:

```json
{
  "id": "1059877",
  "name": "양화한강공원",
  "category": "tourist",
  "categoryName": "관광지",
  "address": "서울특별시 영등포구 노들로 221 (당산동)",
  "lat": 37.5385,
  "lng": 126.8971,
  "image": "https://example.com/image.jpg"
}
```

### 3.1 필드 매핑

| 원본 JSON | API 응답 | 변환 규칙 |
|---|---|---|
| `contentid` | `id` | 문자열 유지 |
| `title` | `name` | 그대로 사용 |
| `contenttypeid` | `category` | 영문 카테고리 코드로 변환 |
| `contentType` | `categoryName` | 한국어 표시명 |
| `addr1`, `addr2` | `address` | 빈 값을 제외하고 결합 |
| `mapy` | `lat` | 문자열을 숫자로 변환 |
| `mapx` | `lng` | 문자열을 숫자로 변환 |
| `firstimage` | `image` | 없으면 `firstimage2`, 둘 다 없으면 빈 문자열 |
| `zipcode` | `zipcode` | 상세 API에서 선택적으로 사용 |
| `cpyrhtDivCd` | `copyrightCode` | 상세 API에서 선택적으로 사용 |
| `tel` | 사용하지 않음 | 화면 요구에 따라 제외 |

원본 JSON에는 상세 설명 필드가 없다. 따라서 프론트의 지역정보 팝업은 이름, 카테고리, 주소, 이미지, 지도 이동 버튼을 중심으로 구성하고 임의 설명을 생성하지 않는다.

## 6. 지역정보 목록·카테고리·검색

### 6.1 지역정보 목록

```http
GET /api/locations?page=1&size=20
```

쿼리 파라미터:

| 이름 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `category` | string | `all` | 지역 카테고리 영문 코드 |
| `search` | string | 빈 문자열 | 장소명 또는 주소 검색어 |
| `page` | integer | 1 | 현재 페이지 |
| `size` | integer | 20 | 페이지당 항목 수 |

서울 데이터는 6,518건이고 쇼핑만 4,368건이므로 서버 페이지네이션을 사용한다. 모든 데이터를 한 번에 프론트로 전달하지 않는다.

### 6.2 카테고리 조회

```http
GET /api/locations?category=shopping&page=1&size=20
```

백엔드는 `shopping`을 TourAPI 코드 `38`로 변환하여 조회한다.

### 6.3 장소·주소 검색

```http
GET /api/locations?search=한강&page=1&size=20
```

검색 대상:

```text
장소명: title
주소: addr1
상세 주소: addr2
```

SQLAlchemy 또는 SQL 검색 조건 예시:

```sql
WHERE title LIKE '%한강%'
   OR addr1 LIKE '%한강%'
   OR addr2 LIKE '%한강%'
```

카테고리와 검색을 함께 사용할 수 있다.

```http
GET /api/locations?category=tourist&search=한강&page=1&size=20
```

빈 결과 응답:

```json
{
  "items": [],
  "total": 0,
  "page": 1,
  "size": 20,
  "totalPages": 0
}
```

여행코스 51건은 원본 주소가 모두 비어 있으므로 주로 장소명으로 검색된다. 주소가 없으면 프론트는 `주소 정보 없음`을 표시한다.

### 6.4 지역정보 카드 클릭

목록 API 응답에 팝업에 필요한 값이 모두 포함되므로 카드 클릭 시 추가 API 없이 이미 받은 데이터로 팝업을 표시하는 방식을 권장한다.

## 7. 지도 API

카테고리별 지도 마커 데이터를 요청한다.

```http
GET /api/locations/map?category=culture
```

응답은 지도에 필요한 최소 필드만 포함한다.

```json
{
  "items": [
    {
      "id": "4077758",
      "name": "노원기차마을",
      "category": "culture",
      "address": "서울특별시 노원구 화랑로 622 (공릉동)",
      "lat": 37.0,
      "lng": 127.0
    }
  ]
}
```

쇼핑 카테고리는 4,368건이므로 모든 마커를 동시에 만들면 브라우저 성능이 저하될 수 있다. 다음 중 하나를 적용한다.

- 마커 최대 개수 제한
- 현재 지도 화면 범위의 데이터만 조회
- 카카오맵 마커 클러스터 사용

간단한 제한 방식:

```http
GET /api/locations/map?category=shopping&limit=500
```

## 10. 게시글 작성·수정·삭제

### 10.1 작성

```http
POST /api/posts
Content-Type: application/json
```

요청:

```json
{
  "category": "관광지",
  "title": "한강공원 후기",
  "author": "익명",
  "content": "야경이 좋았습니다.",
  "password": "1234"
}
```

응답:

```http
HTTP/1.1 201 Created
```

```json
{
  "id": 10,
  "category": "관광지",
  "title": "한강공원 후기",
  "author": "익명",
  "content": "야경이 좋았습니다.",
  "date": "2026-07-15",
  "comments": []
}
```

### 10.2 수정

```http
PUT /api/posts/{postId}
Content-Type: application/json
```

```json
{
  "category": "관광지",
  "title": "수정된 제목",
  "author": "익명",
  "content": "수정된 내용",
  "password": "1234"
}
```

### 10.3 삭제

```http
DELETE /api/posts/{postId}
Content-Type: application/json
```

```json
{
  "password": "1234"
}
```

성공 시:

```http
HTTP/1.1 204 No Content
```

과제 명세에 따라 게시글 수정용 비밀번호는 SQLite에 평문으로 저장하고 수정·삭제 시 문자열 일치 여부로 비교한다. 비밀번호는 API 응답에 포함하지 않는다.

## 12. 챗봇 API

```http
POST /api/chat
Content-Type: application/json
```

요청:

```json
{
  "message": "서울의 한강 관광지를 추천해줘",
  "history": [
    {
      "role": "user",
      "text": "서울 여행을 준비하고 있어"
    }
  ]
}
```

응답:

```json
{
  "message": "한강 관련 관광지로 양화한강공원을 추천합니다."

```

백엔드는 SQLite의 지역정보 또는 게시글을 먼저 검색하고, 검색된 내용을 근거로 GPT 답변을 생성한다. OpenAI API 키는 백엔드 `.env` 파일에 저장하고 Git에서 제외한다.

## 13. 이미지·주소 예외 처리

실제 데이터에서 이미지가 없는 항목 수:

| 카테고리 | 이미지 없음 |
|---|---:|
| 관광지 | 65 |
| 레포츠 | 12 |
| 문화시설 | 39 |
| 쇼핑 | 14 |
| 숙박 | 231 |
| 여행코스 | 6 |
| 축제공연행사 | 2 |

이미지가 없으면 API는 빈 문자열을 반환할 수 있다.

```json
{
  "image": ""
}
```

프론트는 빈 이미지에 LocalHub 기본 이미지를 표시한다.

여행코스 51건은 주소가 모두 비어 있다. 주소가 없으면 다음 문구를 표시한다.

```text
주소 정보 없음
```

확인 결과 서울 데이터의 위도와 경도는 7개 카테고리 모두 제공된다.

## 14. 공통 오류 응답

프론트가 오류 메시지를 바로 표시할 수 있도록 다음 형식으로 통일한다.

```json
{
  "detail": "비밀번호가 일치하지 않습니다."
}
```

| 상태 코드 | 의미 |
|---:|---|
| 200 | 조회·수정 성공 |
| 201 | 작성 성공 |
| 204 | 삭제 성공 |
| 400 | 잘못된 요청 |
| 403 | 비밀번호 불일치 |
| 404 | 게시글·댓글·지역정보 없음 |
| 422 | 필수 입력값 누락 또는 형식 오류 |
| 500 | 서버·DB 오류 |
| 502 | GPT 등 외부 서비스 오류 |
